# Client Engagement Checklist
## Application Portfolio Rationalization

**Version:** 1.0
**Last Updated:** November 2024

---

## Pre-Engagement

### Sales & Scoping
- [ ] Initial discovery call completed
- [ ] Pain points and objectives documented
- [ ] Scope boundaries defined (# of applications, departments, etc.)
- [ ] Timeline expectations aligned
- [ ] Budget range confirmed
- [ ] Decision makers identified
- [ ] Statement of Work drafted
- [ ] SOW signed and PO received

### Client Readiness Assessment
- [ ] Executive sponsor identified and committed
- [ ] Project lead assigned from client side
- [ ] Data sources identified and accessible
- [ ] Stakeholder list preliminary draft completed
- [ ] Known constraints documented (timelines, blackout periods)
- [ ] Previous rationalization attempts reviewed

---

## Phase 1: Initiation (Week 1)

### Kickoff Preparation
- [ ] Kickoff deck prepared with client-specific content
- [ ] Project team introductions prepared
- [ ] Meeting logistics confirmed (location/video, attendees)
- [ ] Pre-reading materials sent to attendees

### Kickoff Meeting
- [ ] Objectives and success criteria reviewed
- [ ] Scope confirmed
- [ ] Timeline and milestones reviewed
- [ ] Roles and responsibilities clarified
- [ ] Communication cadence established
- [ ] Questions and concerns addressed
- [ ] Meeting minutes distributed

### Project Setup
- [ ] Project charter signed
- [ ] SharePoint/collaboration site created
- [ ] Status reporting template established
- [ ] Issue/risk tracking system configured
- [ ] Tool access requested and granted
- [ ] NDA/security requirements completed

### Stakeholder Identification
- [ ] Stakeholder registry created
- [ ] Contact information collected
- [ ] Stakeholder types assigned
- [ ] Application ownership confirmed
- [ ] Interview priority ranking completed

---

## Phase 2: Discovery (Weeks 2-3)

### Data Source Inventory
- [ ] CMDB/asset management system identified
- [ ] Financial systems (licensing, costs) identified
- [ ] APM/monitoring tools identified
- [ ] Service desk/ITSM identified
- [ ] HR systems (FTE data) identified
- [ ] Data owners/contacts for each source confirmed

### Data Collection
- [ ] Data extraction requests submitted
- [ ] CMDB export received and validated
- [ ] Financial data received and validated
- [ ] Usage/performance data received
- [ ] Integration documentation received
- [ ] Support metrics received

### Data Processing
- [ ] Data import completed
- [ ] Deduplication performed
- [ ] Data quality assessment completed
- [ ] Gaps identified and documented
- [ ] Remediation plan for gaps created
- [ ] Master application list established

---

## Phase 3: Stakeholder Assessment (Weeks 3-5)

### Interview Preparation
- [ ] Questionnaire reviewed and customized
- [ ] Interview scheduling process defined
- [ ] Calendar invitations sent
- [ ] Interview guide distributed to team
- [ ] Pre-interview materials sent to stakeholders

### Interview Execution
- [ ] Week 1 interviews completed (target: 10-15)
- [ ] Week 2 interviews completed (target: 10-15)
- [ ] Week 3 interviews completed (target: remaining)
- [ ] All responses entered into assessment tool
- [ ] Missing interviews rescheduled or escalated

### Response Processing
- [ ] All responses validated for completeness
- [ ] Scoring calculations verified
- [ ] Outliers reviewed and explained
- [ ] Coverage report generated
- [ ] Gap list finalized

---

## Phase 4: Analysis (Weeks 5-6)

### Quantitative Analysis
- [ ] Technical health scores calculated
- [ ] Cost efficiency scores calculated
- [ ] Usage metrics normalized
- [ ] Integration complexity scored
- [ ] All quantitative dimensions complete

### Qualitative Integration
- [ ] Assessment scores imported
- [ ] Quantitative and qualitative combined
- [ ] Overall application scores calculated
- [ ] Score distributions reviewed
- [ ] Anomalies investigated

### Disposition Assignment
- [ ] TIME framework applied
- [ ] Tolerate applications identified
- [ ] Invest applications identified
- [ ] Migrate applications identified
- [ ] Eliminate applications identified
- [ ] Edge cases reviewed with leads

### Opportunity Identification
- [ ] Retirement candidates validated
- [ ] Consolidation clusters identified
- [ ] Modernization priorities set
- [ ] Quick wins flagged
- [ ] Savings estimates calculated

---

## Phase 5: Recommendations (Weeks 6-8)

### Recommendation Development
- [ ] Retire recommendations detailed
- [ ] Consolidate recommendations detailed
- [ ] Modernize recommendations detailed
- [ ] Replace recommendations detailed
- [ ] Maintain recommendations detailed

### Roadmap Creation
- [ ] Initiatives sequenced
- [ ] Dependencies mapped
- [ ] Phases defined
- [ ] Quick wins highlighted
- [ ] Resource requirements estimated

### Business Case
- [ ] Savings by initiative calculated
- [ ] Implementation costs estimated
- [ ] ROI calculated
- [ ] Payback period determined
- [ ] Sensitivity analysis performed

### Risk Assessment
- [ ] Risks identified for each recommendation
- [ ] Impact and likelihood assessed
- [ ] Mitigations documented
- [ ] Risk register completed

---

## Phase 6: Validation (Weeks 8-9)

### Validation Sessions
- [ ] IT validation session scheduled
- [ ] Business validation session scheduled
- [ ] Finance validation session scheduled
- [ ] Executive preview scheduled

### Session Execution
- [ ] IT session completed
- [ ] Business session completed
- [ ] Finance session completed
- [ ] All feedback documented

### Refinement
- [ ] Feedback incorporated into recommendations
- [ ] Roadmap adjusted as needed
- [ ] Business case updated
- [ ] Final report compiled

### Executive Materials
- [ ] Executive summary drafted
- [ ] Key findings highlighted
- [ ] Visuals and charts finalized
- [ ] Presentation rehearsed

---

## Phase 7: Delivery (Weeks 9-10)

### Executive Presentation
- [ ] Presentation logistics confirmed
- [ ] All stakeholders invited
- [ ] Presentation delivered
- [ ] Q&A handled
- [ ] Follow-up items captured

### Knowledge Transfer
- [ ] Tool training scheduled
- [ ] Training materials prepared
- [ ] Training session delivered
- [ ] Process documentation delivered
- [ ] Q&A and support period defined

### Project Closure
- [ ] All deliverables submitted
- [ ] Acceptance confirmation received
- [ ] Client satisfaction survey sent
- [ ] Lessons learned documented
- [ ] Final invoice submitted
- [ ] Closeout meeting conducted
